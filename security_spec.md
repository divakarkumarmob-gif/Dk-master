# Security Specification: NEET Prep App

## 1. Data Invariants
- **User Data**: A user can only access their own subcollections (notes, results, mistakes, chat).
- **Community Chat**: Users can send messages with their own `senderId`. Only owners or admins can edit within 5 minutes or delete.
- **Friend Requests**: Only the `toId` can accept (`status` change). Only the `fromId` can mark as seen.
- **Private Chats**: Only `participants` can read/write. Status updates (seen) can only be done by the receiver.
- **Duels**: Only players in the duel can update progress. Anyone can join a 'waiting' duel.
- **Leaderboard**: Users can only update their own ranking.
- **Moderation**: Only admins can modify user moderation fields.

## 2. The "Dirty Dozen" Payloads (Targeting Vulnerabilities)

1. **Privilege Escalation**: User `user-A` tries to read `users/user-B/notes/note-1`.
2. **Identity Spoofing**: User `user-A` creates `community_chat/msg-1` with `senderId: "user-B"`.
3. **Shadow Update**: User `user-A` updates `users/user-A` profile with `moderation: { isBlocked: false }` or `isAdmin: true`.
4. **Relationship Bypass**: User `user-A` creates a `mistake` in `users/user-B/mistakes`.
5. **Update Gap**: User `user-A` updates a `community_chat/msg` owned by `user-B`.
6. **Orphaned Write**: User `user-A` creates a `private_chat` where they are not a participant.
7. **Resource Poisoning**: User `user-A` creates a `note` with a 2MB `text` field (if possible, though Firestore limit is 1MB, we check size).
8. **State Shortcutting**: User `user-A` updates a `friend_request` from `pending` to `accepted` even though they are the `sender`.
9. **Identity Integrity**: User `user-A` updates `leaderboard/user-A` setting `points: 999999`.
10. **PII Leak**: User `user-A` performs a collection group query on `results` to find other users' performance.
11. **Action-Based Update Violation**: User `user-A` updates `private_chat/chat-1` to remove other participants.
12. **System-Generated Bypass**: User `user-A` tries to manually create a document in `study_materials` (Admin only).
